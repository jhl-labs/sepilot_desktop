# SEPilot Desktop v0.7.0 릴리스 노트

**릴리즈 날짜**: 2026-01-04

**SEPilot Desktop v0.7.0**은 v0.5.0 이후 축적된 대규모 업데이트와 안정성 개선을 포함하는 메이저 마일스톤입니다. 단순한 Chat UI를 넘어, **완전한 AI IDE(Integrated Development Environment)**로 진화했습니다.

---

## 🌟 하이라이트 (v0.5.0 이후 주요 변화)

v0.5.0 이후, SEPilot은 다음과 같은 혁신적인 기능들을 도입했습니다:

### 1. 💻 Editor 모드 & IDE 기능

Chat 기반 인터페이스에서 벗어나, 실제 코드를 작성하고 수정할 수 있는 **Editor 모드**가 추가되었습니다.

- **Monaco Editor 통합**: VS Code와 동일한 강력한 코드 편집 경험 제공 (Syntax Highlighting, 미니맵, 다크모드)
- **통합 파일 탐색기**: 프로젝트 파일 트리 뷰, 파일 생성/삭제/이름변경 지원
- **통합 터미널**: xterm.js 기반의 크로스 플랫폼(PowerShell, Bash) 터미널 내장
- **Browser 모드**: 앱 내에서 웹 브라우징, 문서 참조가 가능한 내장 브라우저 (Browser Agent 연동)

### 2. 🧠 RAG 2.0 (검색 증강 생성 대폭 개선)

단순한 벡터 검색을 넘어선 차세대 RAG 엔진을 탑재했습니다.

- **Parent Document Retrieval**: 문서의 맥락을 유지하기 위해 작은 청크로 검색하고 원본 문서를 참조
- **Hybrid Search**: 벡터 유사도(Semantic)와 키워드 매칭(BM25)을 결합하여 검색 정확도 향상
- **Metadata Filtering**: 폴더, 태그, 카테고리 기반의 정교한 필터링 및 부스팅

### 3. 🎨 Creative Tools

- **AI Presentation Lab**: 대화만으로 전문적인 PPT 프레젠테이션 슬라이드 자동 생성 및 내보내기
- **ImageGen 통합**: NanoBanana API 및 ComfyUI를 통한 멀티 프로바이더 이미지 생성 지원
- **Writing Tools**: Notion 스타일의 AI 글쓰기 도구 (요약, 톤 변경, 문법 수정 등)

### 4. 🎭 AI Personas

- **페르소나 시스템**: 번역가, 시니어 개발자, 영어 선생님 등 다양한 전문가 인격 부여
- **Custom Persona**: 사용자 정의 시스템 프롬프트로 나만의 AI 비서 생성

## ✨ v0.7.0 새로운 기능

### 🧜‍♂️ Mermaid 다이어그램 Fallback 시스템

복잡한 다이어그램 생성 시 발생할 수 있는 오류를 자동으로 해결하는 지능형 시스템이 도입되었습니다.

- **자동 문법 수정 (Self-Correction)**: Mermaid 렌더링 실패 시, AI가 자동으로 문법 오류를 분석하고 수정한 뒤 재시도합니다.
- **Text Fallback 모드**: 반복적인 수정 실패 시, 다이어그램을 ASCII Art나 구조화된 텍스트로 자동 변환하여 정보를 손실 없이 전달합니다.
- **향상된 안정성**: 사용자는 더 이상 "Render Error" 메시지를 보지 않고, 항상 시각화된 결과물을 얻을 수 있습니다.

### ⌨️ 인라인 자동완성 토글

사용자의 타이핑 흐름을 방해하지 않는 스마트한 제어 기능이 추가되었습니다.

- **Autocomplete Toggle**: 설정에서 인라인 자동완성(Ghost Text) 기능의 켜고 끄기를 쉽게 제어할 수 있습니다.
- **자연어 처리 강화**: 단순 코드 완성을 넘어, 문맥에 맞는 자연어 문장 자동 완성을 지원합니다.

### 🧩 Extension System (플러그인 아키텍처)

SEPilot Desktop이 진정한 확장 가능한 플랫폼으로 거듭났습니다.

- **VSCode 스타일 Extension 시스템**: VSCode extension과 유사한 플러그인 아키텍처 도입
- **Extension Registry & Loader**: 자동으로 extension을 검색, 로드, 관리하는 시스템
- **동적 로딩**: 앱 시작 시 extensions/ 폴더에서 모든 extension을 자동으로 로드
- **의존성 관리**: Extension 간 의존성 자동 해결 및 순서 관리
- **Manifest 기반 메타데이터**: 각 extension의 정보를 manifest.ts에 선언적으로 정의
- **라이프사이클 훅**: activate/deactivate 훅을 통한 extension 생명주기 관리
- **개발 가이드**: 3rd-party extension 개발을 위한 상세한 가이드 문서 (`extensions/DEVELOPMENT.md`)

**Extension 완전 독립화 (Phase 1 & 2)**:

_Frontend 레이어_:

- `Sidebar.tsx` 완전 동적화: Extension registry 기반으로 모드 선택, 아이콘, 액션 버튼 자동 렌더링
- `BetaSettingsTab.tsx` 완전 동적화: Extension manifest의 betaFlag 기반으로 설정 UI 자동 생성
- `SidebarChat`, `SidebarEditor`, `SidebarBrowser`: 모든 sidebar에서 extension 버튼 동적 생성
- UI 컴포넌트 확장: `HeaderActionsComponent`, `SettingsComponent` 지원
- Lucide React 아이콘 동적 로딩: manifest에 아이콘 이름 문자열로 지정 가능

_Extension 시스템 강화_:

- Extension manifest 확장: `order`, `betaFlag`, `icon`, `ipcChannels` 필드 추가
- `BetaConfig` 동적 타입화: `Record<string, boolean>`로 변경하여 모든 extension betaFlag 지원
- Extension definition 확장: `setupIpcHandlers` 함수 추가로 IPC layer 지원

_IPC 레이어 동적화_:

- Main Process 전용 Extension Loader 추가 (`lib/extensions/loader-main.ts`)
- Presentation IPC handler를 extension으로 완전 분리 (`extensions/presentation/ipc/handlers.ts`)
- `electron/ipc/index.ts`에서 presentation 하드코딩 제거, extension loader로 대체
- Extension manifest에 IPC channel 목록 선언 가능

_Presentation Extension 완전 분리_:

- `PresentationHeaderActions`, `PresentationSettings` 컴포넌트 추가
- IPC handler를 extension 내부로 이동
- Manifest에 `ipcChannels: { handlers: ['presentation:export'] }` 선언

**Editor/Browser Extension 분리 (Phase 3 & 4)**:

_Built-in 기능의 Extension화_:

- Editor 기능을 `extensions/editor/`로 완전 분리
- Browser 기능을 `extensions/browser/`로 완전 분리
- `app/page.tsx`에서 Editor/Browser 하드코딩 제거 - 모든 모드가 extension registry 기반으로 동적 로딩
- Chat을 제외한 **모든 앱 모드가 Extension으로 동작**

_완성된 Extension 생태계_:

- ✅ Editor Extension: Monaco 에디터, 파일 탐색, 터미널, AI 코딩 어시스턴트
- ✅ Browser Extension: 통합 웹 브라우저, AI Agent, 스냅샷, 북마크
- ✅ Presentation Extension: AI 프레젠테이션 생성, PPT/PDF 내보내기

**최종 결과**:

- 신규 extension 추가 시 **메인 앱 코드 0줄 수정**
- Extension 폴더에 파일만 추가하면 UI, IPC, Store, 설정이 모두 자동 등록
- **진정한 플러그인 아키텍처 완성** - VSCode처럼 누구나 extension 개발 가능
- 3rd-party extension 개발을 위한 완벽한 인프라 구축

**Editor Extension 완전 독립화 (Phase 5)**:

_Editor 전용 UI 컴포넌트 분리_:

- `EditorHeaderActions` 컴포넌트 생성 (`extensions/editor/components/EditorHeaderActions.tsx`)
- Sidebar.tsx에서 70+ 줄의 Editor 하드코딩 완전 제거
- AI Chat 토글, Agent/Chat 모드 전환, 파일 탐색기, 검색 버튼 모두 Extension 내부로 이동
- `extensions/editor/index.ts`에서 HeaderActionsComponent export
- Extension Loader에서 EditorHeaderActions 자동 로드 및 등록

_완전한 Extension 독립성 달성_:

- ✅ **Editor Extension**: 메인 앱 코드에 하드코딩 0줄 - 완전 독립
- ✅ **Browser Extension**: 메인 앱 코드에 하드코딩 0줄 - 완전 독립
- ✅ **Presentation Extension**: 메인 앱 코드에 하드코딩 0줄 - 완전 독립
- Sidebar.tsx, app/page.tsx, BetaSettingsTab.tsx 등 모든 메인 앱 컴포넌트가 Extension Registry 기반으로만 동작
- Chat을 제외한 **모든 앱 모드가 100% Extension으로만 동작하는 순수 플러그인 아키텍처 완성**

**Store Layer 동적화 (Phase 6)**:

_Extension Store Slices 통합 시스템_:

- `lib/store/extension-slices.ts` 생성 - 모든 Extension store slice를 중앙 관리
- `chat-store.ts`에서 Presentation 관련 import 완전 제거
- `mergeExtensionStoreSlices()` 함수로 Extension store를 동적으로 병합
- `AppMode` 타입도 Extension manifest 기반으로 자동 생성

_완전한 Extension 독립성 달성 (100%)_:

- ✅ **UI Layer**: 모든 Extension이 완전히 동적 렌더링
- ✅ **Store Layer**: Extension store slice가 extension-slices.ts에서 중앙 관리, chat-store.ts 하드코딩 0줄
- ✅ **Type Layer**: AppMode 타입이 Extension manifest 기반으로 자동 생성
- 신규 Extension 추가 시 **extension-slices.ts 1개 파일**만 수정하면 UI, Store, 타입 모두 자동 등록
- **진정한 Zero-Config Extension System 완성**

**Extension 폴더 구조 완전 독립화 (Phase 7)**:

_Editor/Browser 컴포넌트 Extension 폴더로 완전 이동_:

- `components/editor/` 전체 폴더 (22개 파일) → `extensions/editor/components/`로 이동
- `components/layout/SidebarEditor.tsx` → `extensions/editor/components/`로 이동
- `components/layout/FileExplorer.tsx` → `extensions/editor/components/`로 이동
- `components/layout/FileTreeItem.tsx` → `extensions/editor/components/`로 이동
- `components/layout/FileTreeContextMenu.tsx` → `extensions/editor/components/`로 이동
- **총 Editor 파일 25개** → `extensions/editor/components/`
- `components/browser/` 전체 폴더 (11개 파일) → `extensions/browser/components/`로 이동
- `components/layout/SidebarBrowser.tsx` → `extensions/browser/components/`로 이동
- **총 Browser 파일 12개** → `extensions/browser/components/`
- 모든 Extension이 Presentation과 동일한 자체 완결적 폴더 구조 보유

_완전히 독립적인 Extension 폴더 구조 달성_:

```
extensions/presentation/     extensions/editor/           extensions/browser/
├── components/ (12 files)   ├── components/ (25 files)   ├── components/ (12 files)
├── ipc/                     ├── index.ts                 ├── index.ts
├── lib/                     └── manifest.ts              └── manifest.ts
├── store/
├── types/
├── index.ts
├── manifest.ts
└── README.md
```

- **모든 Extension 코드가 `extensions/` 폴더 내부에만 존재**
- `components/` 폴더에 Extension 관련 코드 **0줄**
- Extension 삭제 시 `extensions/` 폴더만 제거하면 완전 제거 가능
- **진정한 물리적 독립성 달성**

## 🛠 버그 수정 및 개선 사항

- **코드 품질 개선**: ESLint 규칙 준수를 위한 전면적인 코드 리팩토링
  - 중복 타입 정의 제거 (EditorToolsList.tsx)
  - 미사용 import 제거 및 코드 정리
  - 조건문에 curly braces 추가로 가독성 향상
  - Template literal 사용으로 문자열 처리 개선
  - TypeScript const 사용 최적화
- **프로젝트 구조 개선**: RAG 관련 컴포넌트의 일관성 있는 구조화
  - DocumentsPage를 `components/pages/`에서 `components/rag/`로 이동하여 관련 기능(DocumentList, DocumentDialog 등)과 함께 배치
  - 기능별 컴포넌트 그룹화로 코드 가독성 및 유지보수성 향상
- **Ruler Drag Fix**: 에디터의 Word Wrap Ruler가 드래그되지 않던 UI 문제를 해결하여 자유롭게 가이드 선을 조절할 수 있습니다.
- **Agent 안정성 개선**: 긴 작업 수행 시 에이전트가 멈추던 현상(Agent Halt)을 수정하여, 복잡한 태스크도 끝까지 완수하도록 개선했습니다.
- **Electron Stability**: `EPIPE` 에러 등 메인 프로세스와의 통신 불안정 문제를 해결하여 앱의 전반적인 안정성을 높였습니다.
- **성능 최적화**: 대용량 파일 편집 및 렌더링 속도가 최적화되었습니다.
