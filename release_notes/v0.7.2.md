## ✨ 새로운 기능

### 🌍 다국어 지원 확장 (i18n)

UI 전반에 걸쳐 다국어 지원이 대폭 확장되었습니다.

- **WorkingDirectoryIndicator**: 작업 디렉토리 관련 텍스트 국제화
- **Sidebar**: 새 대화 버튼 툴팁 국제화
- **DocsSyncDialog**: GitHub 동기화 설정 전체 UI 국제화
  - Personal Docs / Team Docs 탭 라벨
  - Server Type, GHES URL, Token, Owner, Repository 등 모든 폼 필드
  - 저장/취소/삭제/편집 버튼 및 상태 메시지

### 🛠 Coding Agent 도구 필터링 개선

Thinking 모드에 따른 도구 표시가 개선되었습니다.

- **thinkingMode === 'coding'** 일 때만 builtin 도구(file_read, file_write, file_edit, file_list, command_execute, grep_search) 표시
- 일반 Chat 모드에서는 MCP 서버 도구만 표시하여 UI 간소화

## 🧪 테스트 개선

### 테스트 커버리지 및 유지보수성 향상

전체 테스트 스위트가 대대적으로 검토되고 개선되었습니다.

#### 주요 변경사항

**1. Skip된 테스트 활성화**

- `SettingsDialog.test.tsx`: 전체 테스트 스위트 활성화
- `LLMSettingsTab.test.tsx`: 전체 테스트 스위트 활성화
- `SidebarBrowser.test.tsx`: 전체 테스트 스위트 활성화
- 총 3개의 주요 컴포넌트 테스트가 다시 활성화되어 실행됨

**2. 신규 테스트 추가**

- `UnifiedChatArea.test.tsx`: 새로운 통합 채팅 영역 컴포넌트에 대한 포괄적인 테스트 추가
  - 기본 렌더링 테스트
  - 메시지 표시 테스트
  - 편집/재생성 기능 테스트
  - 폰트 스케일 적용 테스트
  - 다양한 모드(main, browser, editor) 테스트
  - 페르소나 및 스트리밍 상태 테스트

**3. 테스트 문서화**

- `tests/README.md`: 전체 테스트 가이드 작성
  - 테스트 구조 및 실행 방법 상세 설명
  - Jest 설정 및 Mock 파일 가이드
  - 테스트 작성 가이드 및 Best Practices
  - 트러블슈팅 가이드

#### 테스트 구조 개선

- **Frontend 테스트**: JSDOM 환경에서 React 컴포넌트 테스트
- **Backend 테스트**: Node.js 환경에서 라이브러리 로직 테스트
- Mock 파일 체계 정리 및 문서화
- Setup 파일 업데이트로 최신 타입 지원

#### 커버리지 현황

현재 테스트 커버리지 목표:

- Branches: 42%
- Functions: 48%
- Lines: 50%
- Statements: 50%

**4. it.skip 테스트 정리 및 개선**

- **불필요한 테스트 제거**: console.log 테스트 3개 제거
  - `SidebarChat.test.tsx`: Gallery/Settings 버튼 로깅 테스트
  - `SidebarEditor.test.tsx`: Settings 버튼 로깅 테스트

- **Skip 이유 문서화**: 43개의 it.skip 테스트를 카테고리별로 분류
  - Sidebar 구조 변경: 18개 (ChatHistory 컴포넌트로 이동)
  - SSR 환경 제약: 5개 (JSDOM 테스트 불가)
  - 비동기/타이밍 문제: 3개
  - 예외 처리 테스트: 3개
  - 제거된 UI 기능: 4개
  - 기타: 10개

- **테스트 가이드 업데이트**: Skip 정책 및 처리 방침 명확화

**5. E2E (End-to-End) 테스트 추가**

- **Playwright 기반 E2E 테스트 환경 구축**
  - Electron 앱 실행 및 종료를 자동화하는 픽스처 추가
  - 페이지 객체 패턴(POM)을 활용한 유지보수 용이한 테스트 구조
  - 공통 헬퍼 함수 및 유틸리티 제공

- **6가지 주요 테스트 시나리오 구현**
  - TC1: 애플리케이션 실행 및 기본 UI 로딩 테스트
  - TC2: 새 대화 생성 테스트 (단축키, 버튼 클릭)
  - TC3: 메시지 입력 및 전송 테스트 (Enter, Shift+Enter, 특수문자)
  - TC4: 대화 목록 및 네비게이션 테스트 (검색, 이름변경, 삭제)
  - TC5: 설정 페이지 테스트 (섹션 이동, UI/JSON 탭 전환)
  - TC6: 테마 전환 테스트 (라이트/다크 모드)

- **테스트 인프라**
  - 자동 스크린샷 및 비디오 녹화 (실패 시)
  - HTML/JSON/JUnit 리포트 생성
  - 헤드리스/UI 모드 지원
  - 디버그 모드 및 인터랙티브 UI 모드

- **새로운 npm 스크립트**
  - `test:e2e`: 기본 E2E 테스트 실행
  - `test:e2e:headed`: UI 표시 모드로 실행
  - `test:e2e:debug`: 디버그 모드로 실행
  - `test:e2e:ui`: Playwright UI 모드로 실행
  - `test:e2e:report`: 테스트 결과 리포트 보기

- **상세 문서화**
  - `e2e_tests/README.md`: E2E 테스트 가이드 작성
  - 테스트 작성 방법, 실행 방법, 문제 해결 가이드 포함
  - CI/CD 통합 예시 제공
